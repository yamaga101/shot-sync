## v0.2.0 で対応する未済 TODO

### 1. インストール直後 toggle OFF→ON しないと自動 upload が発火しない (UX bug)

**症状**: 初回 install 後、permission を全部許可しても、自動アップロード
toggle を一度 OFF→ON しないと foreground service が起動しない。
初めての user は何が悪いのか分からず詰まる。

**推測される原因**:
- `state.autoSyncEnabled` は DataStore default で `false`
- toggle ON を user が押した時のみ `UploadForegroundService.start()` が呼ばれる
- アプリ初回起動では service が起動するパスが無い

**修正案**:
- App 起動時 (MainViewModel.init) で `autoSyncEnabled && permissions.allGreen`
  ならば `UploadForegroundService.start(context)` を idempotent に呼ぶ
- BootReceiver は既に同様のロジックを持っている (`autoSyncEnabled.first()`
  → `UploadForegroundService.start`) ので、参考にする
- もしくは、auto-sync を default ON にして permission allGreen を待つ
  pattern にする (より aggressive)
- 通知バーに「監視中」が出てない時に状態を warn する UI (診断 + 起動 trigger)

### 2. Camera 用 Drive folder default を設定

URL: https://drive.google.com/drive/folders/12y4EpdE_jXuXbi6E0rxpqUblA4t3rCLp

**実装**:
- `Settings.kt` の `DEFAULT_DRIVE_FOLDER_ID` (Screenshots 用、`1DBOVz...`) と
  並ぶ形で `DEFAULT_CAMERA_DRIVE_FOLDER_ID = "12y4EpdE_jXuXbi6E0rxpqUblA4t3rCLp"`
  を追加
- `cameraDriveFolderId` Flow / snapshot で未設定時に default を返す
- UI placeholder は「(default: 12y4EpdE...)」と説明
- 既に空欄保存している user は default 反映、保存済みの user は touch しない
