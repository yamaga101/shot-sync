(1) 揃えるのは「見つけ方」だけで、位置は platform 慣習優先。Android は設定/About、拡張は popup/options footer、PWA は設定/フッター。表示は `vX.Y.Z (sha, type)` が実用的。build date は「最新判定」には便利だが再現性を濁すので補助扱い。SoT は各技術の公式 manifest/Gradle/package に置き、定数は生成。規約は `standards.md`、bump 手順は `operating.md`。

(2) 推奨は A。2 つ目の folder ID を明示し、空なら現行 folder に混在で後方互換。B は親 folder 権限、同名 subfolder、削除済み再作成で事故りやすい。C は視覚的に弱く、D は Drive UI で見えず API 依存。edge case は folder 削除/権限剥奪/ID 重複/空設定を起動時または upload 前に検査し、勝手に別場所へ逃がさず警告が堅い。
